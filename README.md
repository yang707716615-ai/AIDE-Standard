## 🎯 一句话介绍 / One Sentence Intro

**中文**：AIDE（AI Design Employee）是一套跨行业的AI员工设计标准，让你能像写岗位说明书一样，设计、评估、管理你的AI员工团队。

**English**: AIDE is a cross-industry standard for designing AI employees, enabling you to design, evaluate, and manage AI teams like writing job descriptions.

> 别人教你怎么写Prompt，AIDE教你怎么设计完整的AI员工体系。
> 
> *Don't just write prompts, design complete AI employee systems with AIDE.*

## 🏗️ AIDE四大支柱 / Four Pillars

| 支柱 / Pillar | 解决的问题 / Problem Solved | 类比 / Analogy |
|--------------|---------------------------|----------------|
| **RACI-D** | 人和AI各自该做什么？<br>What should humans vs. AI do? | 公司岗位职责表<br>Company job description |
| **STATE** | 什么时候自动做？什么时候停？<br>When to automate? When to pause? | 工厂生产线流程<br>Factory production line |
| **IOCA** | 怎么输入？输出什么标准？<br>How to input? What output standards? | 餐厅点菜-做菜-上菜标准<br>Restaurant order-cook-serve standard |
| **MBVE** | 怎么越用越聪明？怎么评估价值？<br>How to get smarter? How to measure value? | 员工试用期+晋升体系<br>Employee probation + promotion system |

## 🎯 为什么需要AIDE？

### 对比现有方案

| 方案 | 问题 | AIDE优势 |
|------|------|----------|
| **Prompt技巧** | 碎片化，没有体系 | 结构化设计流程 |
| **AI工具教程** | 只教怎么用工具 | 教怎么设计AI员工体系 |
| **技术框架**<br>(ReAct/CoT) | 技术人员专用 | 业务语言，零基础可用 |
| **雇佣提示工程师** | 贵，不可持续 | 一次学习，终身受用 |

### AIDE的核心价值

1. **标准化**：像招聘员工一样标准化设计AI
2. **可评估**：有明确的质量标准和ROI计算
3. **可进化**：从L1新手级进化到L3专家级
4. **跨行业**：无论视频/电商/写作/客服，同一套标准

## 📖 深入理解AIDE

### AIDE的理论基础

AIDE不是凭空创造，而是融合了多个成熟领域的最佳实践：

- **RACI-D** → 来自项目管理RACI矩阵
- **STATE** → 来自软件工程状态机
- **IOCA** → 来自系统论和控制论
- **MBVE** → 来自组织学习理论

### 与技术框架的关系

```
技术层（技术人员使用）：
ReAct/CoT/ToT → 实现STATE状态流转的底层逻辑

业务层（普通人使用）：
AIDE标准 → 用业务语言设计AI员工，无需懂技术

关系：
AIDE标准可以运行在ReAct/CoT等技术框架之上
普通人用AIDE设计，技术人员用ReAct实现
```
