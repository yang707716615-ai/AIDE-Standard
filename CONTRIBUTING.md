# 贡献指南

感谢您对AIDE Standard的兴趣！我们欢迎各种形式的贡献。

---

## 如何贡献

### 1. 提交案例

如果您用AIDE标准设计了AI员工，欢迎分享：

**提交方式**：
1. Fork本仓库
2. 在 `/templates/examples/` 目录下创建您的案例文件
3. 参考现有案例格式（如 `example-video-creator.md`）
4. 提交Pull Request

**案例要求**：
- 使用AIDE四大支柱（RACI-D/STATE/IOCA/MBVE）
- 有完整的设计过程
- 有实际使用效果数据（可选但推荐）
- 注明适用场景

---

### 2. 改进标准

如果您对AIDE框架有改进建议：

**提交方式**：
1. 提交Issue，描述您的建议
2. 说明改进的理由和预期效果
3. 如果可能，提供改进后的示例

**我们关注的改进方向**：
- 让框架更易懂
- 增加新的评估维度
- 优化填写模板
- 补充行业特定指南

---

### 3. 翻译文档

帮助将AIDE标准翻译成其他语言：

**目标语言**：
- 英文（优先）
- 日文
- 韩文
- 其他

**翻译要求**：
- 保持专业术语准确
- 适应当地文化（必要时）
- 保持格式一致

---

### 4. 修复错误

发现文档中的错误：

**包括**：
- 错别字
- 格式错误
- 链接失效
- 内容过时

**提交方式**：
- 小问题：直接提交Pull Request
- 大问题：先提交Issue讨论

---

## 提交规范

### Pull Request规范

**标题格式**：
```
[类型] 简要描述

类型包括：
- [案例] 提交新案例
- [文档] 改进文档
- [翻译] 翻译文档
- [修复] 修复错误
- [其他] 其他贡献
```

**描述要求**：
1. 说明做了什么修改
2. 为什么要做这个修改
3. 测试/验证情况
4. 相关Issue编号（如有）

### 案例提交模板

```markdown
## 案例名称：[AI员工名称]

### 适用场景
[描述适用的人群和场景]

### 设计亮点
[这个设计的特别之处]

### 使用效果
- 节省时间：X小时/天
- 产量提升：X%
- ROI：X%

### 作者
- 姓名：[您的名字]
- 联系方式：[可选]
- 使用时长：X个月
```

## 行为准则

### 我们的承诺

为了营造开放和友好的环境，我们作为贡献者和维护者承诺：

- 尊重不同的观点和经验
- 接受建设性的批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

### 不可接受的行为

- 使用带有性暗示的语言或图像
- 挑衅、侮辱或贬低性评论
- 公开或私下的骚扰
- 未经明确许可发布他人的私人信息
- 其他不道德或不专业的行为

## 开发流程

### 分支管理

- `main`：稳定版本
- `develop`：开发版本
- `feature/*`：新功能分支
- `fix/*`：修复分支

### 提交信息规范

```
类型: 简短描述

详细说明（可选）

相关Issue: #123
```

**类型包括**：
- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `style`: 格式（不影响代码含义的更改）
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建过程或辅助工具的变动
