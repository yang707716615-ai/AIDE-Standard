### 第2条（痛点）

```
【AIDE解决什么问题？】

你是不是也遇到过这些情况：

• 买了ChatGPT/Claude会员，但不知道问什么
• AI回答质量不稳定，时好时坏
• 想自动化工作流，但担心AI出错
• 团队想用AI提效，但不知道怎么分工

这些问题的本质是：
我们把AI当"工具"用，而不是当"员工"管。

AIDE标准的核心就是：
像招聘、培训、管理真实员工一样，
设计你的AI员工团队。
```

---

### 第3条（四大支柱）

```
【AIDE四大支柱】

1️⃣ RACI-D｜职责定义
就像公司里的岗位说明书，
明确人和AI各自该做什么。

2️⃣ STATE｜状态机
就像工厂生产线，
什么时候自动跑，什么时候停下来检查。

3️⃣ IOCA｜输入输出标准
就像餐厅的点菜-做菜-上菜标准，
确保每次输出质量一致。

4️⃣ MBVE｜记忆与进化
就像员工试用期+晋升体系，
让AI越用越聪明，从新手变成专家。

完整文档在GitHub /docs 目录，5-10分钟读完一个标准。
```

### 第5条（如何使用）

```
【3步上手，不用写代码】

第1步：选一个案例（1分钟）
从 /templates/examples/ 选一个最接近你需求的

第2步：填空模板（5分钟）
打开 /templates/template-blank.md 照着填

第3步：配置到AI工具（1分钟）
把填好的内容配置到：
• 扣子 / Coze
• Dify
• OpenClaw
• 任何支持自定义Prompt的AI平台

7分钟，你的AI员工就能上岗。
```
