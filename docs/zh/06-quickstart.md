# 06 - 快速开始指南

> **目标**：5分钟上手，设计出你的第一个AI员工  
> **前提**：已经阅读AIDE四大支柱（或理解基本概念）

---

## 只需3步

### 第1步：选一个案例（1分钟）

从我们准备好的案例库，选一个最接近你需求的：

```
/templates/examples/
├── example-video-creator.md      ⭐ 视频创作AI（推荐新手）
├── example-ecommerce-operator.md   电商运营AI
├── example-content-writer.md       内容写作AI
└── example-customer-service.md     客服AI
```

**推荐新手从"视频创作AI"开始**，因为：
- 流程完整（覆盖所有STATE状态）
- 容易理解（大家都刷过短视频）
- 效果可见（做完就能发视频）

---

### 第2步：填空模板（3分钟）

打开空白模板，照着案例填：

```bash
# 打开空白模板
/templates/template-blank.md

# 不知道咋填？看填写指南
/templates/template-guide.md
```

**填空顺序建议**：

1. **填RACI-D**（1分钟）
   - 列出你的3个核心任务
   - 分配R/A/C/I/D

2. **画STATE流程**（1分钟）
   - 画出工作流
   - 标出AUTO/WAIT/CHECK/HALT

3. **写IOCA**（1分钟）
   - 输入：需要什么信息？
   - 输出：交付什么？质量标准？
   - 控制：检查点在哪？
   - 适配：怎么越用越聪明？

4. **设计MBVE**（可选，后期优化）
   - 先跳过，用默认配置
   - 使用10次后再优化

---

### 第3步：应用到AI工具（1分钟）

把填好的内容，配置到你的AI平台：

**支持的平台**：
- [扣子](https://coze.cn)（推荐，国内友好）
- [Coze国际版](https://coze.com)
- [Dify](https://dify.ai)
- [OpenClaw](https://github.com/claw-ai/claw)（开源）
- 任何支持自定义Prompt的AI工具

**配置方法**：
1. 把RACI-D转换成系统Prompt（告诉AI它的职责）
2. 把STATE画成工作流图（在平台上配置节点）
3. 把IOCA写成输入输出格式定义
4. 测试运行，调优

**完成！** 你的AI员工上岗了 🎉

## 完整示例：5分钟设计视频创作AI

### 第1步：选案例
选 `example-video-creator.md`

### 第2步：填空

```markdown
# 我的AI员工：短视频创作助手

## RACI-D职责分配

### 任务1：每日选题
- R（人类执行）：最终选择1个选题
- A（人类负责）：对选题效果负责
- C（AI咨询）：AI推荐5个选题+数据
- I（AI知情）：AI记录历史数据
- D（AI委托）：AI监控热点榜单

### 任务2：生成提示词
- R（人类执行）：确认最终提示词
- A（人类负责）：对视频质量负责
- C（AI咨询）：AI生成3版供选择
- I（AI知情）：AI保存版本历史
- D（AI委托）：AI优化提示词

### 任务3：撰写文案
- R（人类执行）：发布前审核文案
- A（人类负责）：对文案合规负责
- C（AI咨询）：AI生成4平台版本
- I（AI知情）：AI记录文案效果
- D（AI委托）：AI适配平台风格

## STATE状态机

```
[IDLE] 每天8:00自动启动
  ↓
[AUTO] AI扫描热点，推荐5个选题
  ↓
[WAIT] 暂停，等人类选择（30秒超时）
  ↓
[AUTO] AI生成3版提示词
  ↓
[WAIT] 暂停，等人类确认
  ↓
[CHECK] AI自检评分>85分？
  ↓
[AUTO] AI生成4平台文案
  ↓
[WAIT] 暂停，等人类确认
  ↓
[HALT] AI无法直接生成视频，给操作指南
  ↓
[IDLE] 完成任务
```

## IOCA输入输出

### 输入
- 固定：热点榜单（每小时）
- 触发：每天8:00/用户指令
- 人工：创作方向、特殊要求

### 输出
- 选题列表（表格，5个选项）
- 提示词（文本，中英双语，3版）
- 文案（文本，4平台版本）

### 控制
- 事前：敏感词过滤
- 事中：质量检查>85分
- 事后：用户评分

### 适配
- 上下文：记住本次偏好
- 用户偏好：常选题材权重+30%
- 场景：工作日vs周末不同推荐
```

### 第3步：应用到扣子

**在扣子平台配置**：

1. **创建Bot**：名字叫"视频创作助手"

2. **写系统Prompt**（基于RACI-D）：
```
你是专业的短视频创作助手。

你的职责：
- 每天推荐热点选题
- 生成视频提示词
- 撰写多平台文案

你可以做的（D/C/I）：
- 自动监控热点榜单
- 推荐选题并给出数据
- 生成提示词和文案
- 记录和分析数据

你必须让人类做的（R/A）：
- 最终选择选题
- 确认提示词
- 审核文案
- 点击发布按钮

工作流程：
1. 每天8点，自动推荐5个选题
2. 等待人类选择
3. 生成3版提示词
4. 等待人类确认
5. 生成4平台文案
6. 等待人类确认
7. 给出视频生成操作指南

质量标准：
- 提示词必须包含：主体+动作+场景+风格+镜头+特效
- 评分必须>85分
- 文案必须适配各平台风格
```

3. **配置工作流**（基于STATE）：
- 节点1：定时触发（每天8:00）→ AUTO
- 节点2：推荐选题 → WAIT（等用户选择）
- 节点3：生成提示词 → WAIT（等用户选择）
- 节点4：质量检查 → CHECK
- 节点5：生成文案 → WAIT（等用户确认）
- 节点6：完成 → IDLE

4. **测试运行**
   - 发送："今天想做武侠题材"
   - 看AI是否按STATE流程执行
   - 调优直到满意

## 常见问题

**Q：我不会用扣子，怎么办？**

A：
1. 先看扣子的官方教程（30分钟）
2. 或者先用ChatGPT/Claude测试你的Prompt
3. 等熟悉后再迁移到扣子

**Q：填完模板后，AI还是做得不好，怎么办？**

A：
1. 检查RACI-D：是不是给AI的任务太模糊？
2. 检查STATE：是不是缺少CHECK节点？
3. 检查IOCA：输出标准是否清晰？
4. 使用[评估检查清单](/checklists/evaluate-checklist.md)诊断

**Q：我想设计其他类型的AI员工，有参考吗？**

A：
- 看 `/templates/examples/` 里的其他案例
- 电商运营、内容写作、客服，都有完整示例

**Q：设计完成后，怎么评估好不好？**

A：
使用QUICK评估：
- Q（质量）：输出是否符合预期？1-10分
- U（可用性）：是否可以直接用？1-10分
- I（智能度）：是否体现AI价值？1-10分
- C（一致性）：多次执行是否稳定？1-10分
- K（知识度）：是否记住了偏好？1-10分

总分>40分（平均8分）：优秀，可以正式使用
总分30-40分：良好，需要微调
总分<30分：需要重新设计
