# 05 - MBVE标准（记忆-行为-价值-进化）

> **阅读时间**：10分钟  
> **目标**：学会让AI员工越用越聪明，并评估其商业价值

---

## 什么是MBVE？

**MBVE** = **M**emory（记忆）+ **B**ehavior（行为）+ **V**alue（价值）+ **E**volution（进化）

**核心作用**：
> 让AI员工能记住、守规矩、创造价值，并从新手成长为专家。

**类比**：员工的试用期考核 + 晋升体系

---

## 1. Memory（记忆系统）

### 三层记忆架构

| 层级 | 时长 | 容量 | 存储内容 | 示例 |
|------|------|------|----------|------|
| **短期记忆（STM）** | 当前对话 | 20轮 | 对话上下文 | 用户刚说"要反转剧情" |
| **中期记忆（MTM）** | 本次工作流 | 7天 | 本次任务数据 | 本周生成的所有提示词 |
| **长期记忆（LTM）** | 永久 | 无限 | 用户画像、历史案例 | 用户偏好"武侠"题材 |

### 记忆应用示例

```markdown
## 视频创作AI的记忆系统

### 短期记忆（当前对话）
- 用户3分钟前说："今天想做武侠题材"
- 用户1分钟前说："要有反转剧情"
- AI当前生成策略：优先推荐武侠+反转

### 中期记忆（本周）
- 本周已生成：5个武侠题材提示词
- 本周最高播放："武林高手面试记"（50万播放）
- AI优化策略：多生成类似风格

### 长期记忆（永久）
- 用户画像：
  - 偏好题材：武侠（70%）、职场（20%）、其他（10%）
  - 偏好风格：搞笑反转
  - 活跃时间：晚上8-10点
  - 质量要求：高（很少选AI的第一推荐）
- 成功案例库：
  - 高播放案例：10个（提示词存档）
  - 失败案例：3个（避免重复）
```

### 记忆的作用

```
没有记忆：
用户："今天想做武侠题材"
AI：推荐5个选题（随机）
用户选A
...
第二天
用户："今天想做武侠题材"
AI：推荐5个选题（还是随机，和昨天一样）

有记忆：
用户："今天想做武侠题材"
AI：推荐5个选题（根据昨天选择的A，推荐类似风格）
用户选B
AI记录：用户连续2天选武侠，且偏好反转剧情
...
第七天
AI主动："根据你的历史偏好，本周武侠题材有新热点，推荐..."
```

## 2. Behavior（行为规范）

### 四大约束

| 规范 | 定义 | 示例 |
|------|------|------|
| **边界意识** | 明确知道什么能做，什么不能做 | AI不直接发布视频（必须人工） |
| **一致性** | 相同输入，相同场景，输出稳定 | 同样"武侠"选题，风格统一 |
| **可预测** | 人类能预判AI的行为 | 知道AI会在哪些节点暂停 |
| **可追溯** | 每个决策都有据可查 | 能查看AI为什么推荐这个选题 |

### 行为准则（B-标准）

```markdown
## 视频创作AI的行为规范

### 边界意识
✅ 能做的：
- 推荐选题
- 生成提示词
- 撰写文案
- 数据分析

❌ 不能做的：
- 直接发布视频（涉及账号安全）
- 承诺播放效果（无法保证）
- 绕过人工确认（关键节点必须WAIT）

### 一致性
- 用户偏好"武侠"，AI持续推荐武侠
- 用户要求"反转"，AI每次生成都有反转元素
- 输出格式统一（中英双语提示词）

### 可预测
- 人类知道：选题阶段AI会WAIT
- 人类知道：生成提示词后AI会CHECK
- 人类知道：发布前AI会HALT

### 可追溯
- 推荐"武林高手面试记"，AI给出理由：
  - 原因1：用户历史偏好武侠（LTM）
  - 原因2：当前热点榜单TOP3（固定输入）
  - 原因3：类似选题历史播放50万+（MTM）
```

## 3. Value（价值评估）

### ROI计算公式

```
AI员工ROI = (节省时间价值 + 创造价值 - AI成本) / AI成本 × 100%

其中：
节省时间价值 = (人工耗时 - AI耗时) × 人工时薪 × 任务频次
创造价值 = 质量提升收益 + 速度提升机会收益
AI成本 = 工具订阅费 + 配置时间成本 + 学习成本
```

### 计算示例：视频创作AI

```markdown
## ROI计算：视频创作AI员工

### 传统人工方式
- 制作1条视频耗时：3小时
- 时薪：¥100
- 单条成本：¥300
- 月产量：10条
- 月成本：¥3000

### AI员工方式
- AI辅助制作耗时：30分钟
- 人工时薪：¥100（节省2.5小时）
- AI工具订阅：¥200/月
- AI调用成本：¥0.5/条 × 30条 = ¥15
- 初期配置：¥500（一次性）
- 学习成本：¥0（使用AIDE标准）

### 计算
节省时间价值 = 2.5小时 × ¥100 × 30条 = ¥7500/月
创造价值：
  - 产量提升：从10条→30条，多20条
  - 假设每条平均收益¥200
  - 额外收益 = 20条 × ¥200 = ¥4000/月
AI成本 = ¥200 + ¥15 = ¥215/月

首月ROI = (7500 + 4000 - 215 - 500) / (215 + 500) = 10785 / 715 = 1508%
次月ROI = (7500 + 4000 - 215) / 215 = 11285 / 215 = 5248%

结论：首月ROI 1508%，次月ROI 5248%
```

### QUICK质量评估模型

除了ROI，还有质量评估：

```
Q - Quality（质量）：输出是否符合预期？ 1-10分
U - Usability（可用性）：是否可以直接使用？ 1-10分
I - Intelligence（智能度）：是否体现AI价值？ 1-10分
C - Consistency（一致性）：多次执行是否稳定？ 1-10分
K - Knowledge（知识度）：是否利用了记忆？ 1-10分

总分 = (Q + U + I + C + K) / 5

8-10分：优秀，可以直接交付
6-8分：良好，轻微修正后可用
4-6分：及格，需要较多修正
<4分：不合格，重新设计
```

## 4. Evolution（进化机制）

### 三阶段进化

| 等级 | 名称 | 特征 | 人机比例 | 适用场景 |
|------|------|------|----------|----------|
| **L1** | 新手级 | 每个节点都需确认 | 人工70% : AI 30% | 首次使用、低风险容忍 |
| **L2** | 熟练级 | 部分节点自动 | 人工30% : AI 70% | 有信任基础、中等复杂度 |
| **L3** | 专家级 | 全自动，仅异常时求助 | 人工5% : AI 95% | 高度信任、标准化场景 |

### 进化路径示例

```markdown
## 视频创作AI的进化路径

### Level 1（第1-10条视频）
- **特征**：每个WAIT都停，每个CHECK都人工看
- **流程**：
  - [WAIT] AI推荐5个选题 → 人类选择
  - [WAIT] AI生成3版提示词 → 人类选择
  - [CHECK] AI自检评分 → 人类确认
  - [WAIT] AI生成文案 → 人类修改
  - [HALT] 发布前 → 人类最终确认
- **耗时**：人类15分钟/条

### Level 2（第11-50条视频）
- **特征**：提示词生成自动通过，只在选题和发布时WAIT
- **变化**：
  - [AUTO] 提示词生成（质量稳定，自动通过）
  - [CHECK] 自检不通过时，自动重试（最多3次）
  - [WAIT] 只在选题和发布时暂停
- **耗时**：人类5分钟/条
- **触发条件**：
  - 使用满10次
  - CHECK通过率>90%
  - 用户评分>4星

### Level 3（第50+条视频）
- **特征**：全自动推荐+生成，人类只最终确认
- **变化**：
  - [AUTO] 全自动选题（基于LTM偏好）
  - [AUTO] 全自动生成提示词
  - [AUTO] 全自动生成文案
  - [WAIT] 只在发布前最终确认
- **耗时**：人类1分钟/条（只看一眼）
- **触发条件**：
  - 使用满50次
  - L2阶段CHECK通过率>95%
  - 连续20次无人工修改

### 降级机制
如果L3阶段出现：
- 连续3次CHECK不通过 → 降级到L2
- 用户评分<3星 → 降级到L1
```

## 完整示例：视频创作AI的MBVE设计

```markdown
# MBVE设计文档：视频创作AI员工

## Memory（记忆）

### 短期记忆（当前对话）
- 保存：用户本次提到的偏好、要求
- 容量：20轮对话
- 应用：实时调整当前生成策略

### 中期记忆（本周）
- 保存：本周生成的所有内容、效果数据
- 容量：7天
- 应用：本周内优化生成策略

### 长期记忆（永久）
- 用户画像：偏好题材、风格、活跃时间、质量要求
- 成功案例库：高播放案例的提示词和策略
- 失败案例库：低播放案例，避免重复

## Behavior（行为）

### 边界
- ✅ 能：推荐、生成、分析
- ❌ 不能：直接发布、承诺效果

### 一致性
- 输出格式统一
- 风格与历史偏好一致

### 可预测
- 明确的状态流转
- 人类知道何时需要参与

### 可追溯
- 每个推荐都有理由
- 数据来源可查

## Value（价值）

### ROI
- 节省时间：2.5小时/条
- 创造价值：额外产量收益
- 首月ROI：1508%

### QUICK评分
- Q（质量）：9/10
- U（可用性）：8/10
- I（智能度）：8/10
- C（一致性）：9/10
- K（知识度）：7/10
- 总分：8.2/10 ✅ 良好

## Evolution（进化）

当前等级：L1（第1-10条）
升级条件：使用10次 + CHECK通过率>90%
目标等级：L3（全自动）
预计时间：使用50次后（约2个月）
```

## 自检清单

设计MBVE时，检查：

- [ ] 有三层记忆（短期/中期/长期）
- [ ] 记忆有明确的应用场景
- [ ] 有行为边界（能/不能清单）
- [ ] 有ROI计算模型
- [ ] 有QUICK质量评估
- [ ] 有三级进化路径
- [ ] 有升级和降级机制
