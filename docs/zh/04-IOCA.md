# 04 - IOCA标准（输入输出控制架构）

> **阅读时间**：10分钟  
> **目标**：学会设计AI员工的输入输出，建立质量控制机制

---

## 什么是IOCA？

**IOCA** = **I**nput（输入）+ **O**utput（输出）+ **C**ontrol（控制）+ **A**daptation（适配）

**核心作用**：
> 定义AI员工怎么接收信息、怎么交付结果、怎么保证质量、怎么适应不同场景。

**类比**：餐厅的点菜-做菜-上菜标准流程

---

## 1. Input（输入设计）

### 三种输入类型

| 类型 | 定义 | 示例 | AI角色 |
|------|------|------|--------|
| **固定输入** 📋 | AI自动获取的 | 热点榜单、历史数据 | 自动抓取 |
| **触发输入** 🔔 | 事件驱动的 | 用户指令、定时任务 | 监听触发 |
| **人工输入** ✍️ | 人类提供的 | 创作方向、特殊要求 | 接收处理 |

### 输入质量标准（I-标准）

```
I1 - 完整性：必需信息是否齐全？
I2 - 准确性：信息来源是否可靠？
I3 - 及时性：信息是否最新？
I4 - 格式规范：是否符合AI处理格式？
```

### 示例：视频创作AI的输入设计

```markdown
## 输入清单

### 固定输入（AI自动获取）
- [ ] 每日热点榜单（抖音热榜、微博热搜）
- [ ] 历史视频数据（播放量、互动率）
- [ ] 竞品账号动态（可选）
- 来源可靠性：⭐⭐⭐⭐⭐
- 更新频率：每小时

### 触发输入（事件驱动）
- [ ] 用户指令："今天想做武侠题材"
- [ ] 定时任务：每天早上8:00自动启动
- [ ] 热点突变：某话题突然爆火（阈值：1小时内增长>1000%）

### 人工输入（用户提供）
- [x] 创作方向偏好：武侠、搞笑、职场
- [x] 特殊要求："要有反转"
- [x] 禁止事项："不要出现暴力"
- 输入格式：自然语言/关键词/选项
```

## 2. Output（输出设计）

### 三个维度

| 维度 | 问题 | 示例 |
|------|------|------|
| **格式** | 输出什么形式？ | 文本/表格/图片/视频/代码 |
| **标准** | 质量要求是什么？ | 评分>80分/原创度>90% |
| **交付** | 怎么交付？ | 即时推送/人工确认后发送 |

### 输出质量标准（O-标准）

```
O1 - 准确性：事实正确，无错误
O2 - 完整性：覆盖所有要求
O3 - 可用性：直接可用，无需二次加工
O4 - 一致性：风格、格式统一
```

### 示例：视频创作AI的输出设计

```markdown
## 输出清单

### 输出1：选题推荐列表
- **格式**：Markdown表格
- **标准**：
  - 至少5个选项
  - 每个选项包含：标题+预估播放量+难度
  - 数据时效性<24小时
- **交付**：即时推送，WAIT状态等待选择

### 输出2：视频提示词
- **格式**：文本（中英双语）
- **标准**：
  - 包含：主体+动作+场景+风格+镜头+特效
  - 质量评分>85分（CHECK自检）
  - 原创度>80%（不与历史重复）
- **交付**：生成3版，WAIT状态等待选择

### 输出3：多平台文案
- **格式**：文本（分平台版本）
- **标准**：
  - 抖音版：15秒，快节奏，强钩子
  - 小红书版：图文配合，emoji，干货
  - 视频号版：故事性，中老年友好
  - B站版：深度，年轻向，弹幕互动
- **交付**：WAIT状态等待确认

### 输出4：数据报表
- **格式**：Excel/图表
- **标准**：
  - 包含：播放量、点赞、评论、转化率
  - 可视化：趋势图、对比图
  - 分析：AI自动给出优化建议
- **交付**：每周一自动发送（I状态）
```

## 3. Control（控制机制）

### 三层控制

| 层级 | 时机 | 手段 | 示例 |
|------|------|------|------|
| **事前控制** 🔒 | 输入阶段 | 校验、过滤、权限检查 | 敏感词过滤、输入完整性检查 |
| **事中控制** ⚡ | 处理阶段 | 进度监控、质量检查点 | 30秒超时、质量评分检查 |
| **事后控制** 📊 | 输出阶段 | 结果评估、反馈循环 | 用户评分、效果追踪 |

### 控制点设计示例

```markdown
## 视频创作AI的控制机制

### 事前控制（输入阶段）
- [ ] 敏感词检测：输入内容是否包含违禁词？
- [ ] 完整性检查：必需信息是否齐全？
- [ ] 权限检查：用户是否有足够额度？

### 事中控制（处理阶段）
- [ ] 超时控制：每个AUTO节点不超过3分钟
- [ ] 质量检查点：提示词生成后CHECK自检
- [ ] 异常监控：连续3次失败自动HALT

### 事后控制（输出阶段）
- [ ] 用户评分：1-5星评价
- [ ] 效果追踪：发布后24小时追踪数据
- [ ] 反馈循环：根据效果优化AI模型
```

## 4. Adaptation（适配机制）

### 三种适配

| 适配类型 | 适配对象 | 机制 | 示例 |
|---------|---------|------|------|
| **上下文适配** 🔄 | 当前对话 | 根据对话历史调整 | 记住用户3分钟前提到的偏好 |
| **用户偏好适配** 👤 | 长期习惯 | 学习用户历史选择 | 记住用户常选"武侠"题材 |
| **场景适配** 🎭 | 外部环境 | 根据场景调整策略 | 工作日vs周末推荐不同节奏 |

### 适配示例

```markdown
## 视频创作AI的适配机制

### 上下文适配
- 当前对话已提到"想要反转剧情"
- AI调整：后续生成优先推荐有反转的选题

### 用户偏好适配（长期学习）
- 历史记录：用户过去10次选择中，7次选"武侠"
- AI调整：推荐列表中武侠题材权重+30%

### 场景适配
- 工作日（周一至周五）：推荐职场、效率类选题
- 周末（周六至周日）：推荐娱乐、搞笑类选题
- 节日（如春节）：推荐节日相关选题
```

## 完整示例：视频创作AI的IOCA设计

```markdown
# IOCA设计文档：视频创作AI员工

## Input（输入）

### 固定输入
| 数据来源 | 内容 | 频率 | 可靠性 |
|---------|------|------|--------|
| 抖音热榜 | 热门话题、音乐、挑战 | 每小时 | ⭐⭐⭐⭐⭐ |
| 微博热搜 | 社会热点、娱乐新闻 | 实时 | ⭐⭐⭐⭐ |
| 历史数据库 | 过往视频表现数据 | 实时 | ⭐⭐⭐⭐⭐ |

### 触发输入
| 触发条件 | 动作 |
|---------|------|
| 用户指令："今天想做XX" | 启动创作流程 |
| 定时：每天8:00 | 自动推荐选题 |
| 热点突变：1小时内增长>1000% | 紧急推荐 |

### 人工输入
| 输入项 | 格式 | 示例 |
|--------|------|------|
| 创作方向 | 关键词 | "武侠、搞笑、职场" |
| 特殊要求 | 自然语言 | "要有反转，不要太长" |
| 禁止事项 | 列表 | "不要暴力、不要政治" |

## Output（输出）

| 输出项 | 格式 | 标准 | 交付方式 |
|--------|------|------|----------|
| 选题推荐 | 表格 | 5个选项+数据 | WAIT，等选择 |
| 视频提示词 | 文本 | 评分>85，3版 | WAIT，等确认 |
| 多平台文案 | 文本 | 4平台版本 | WAIT，等确认 |
| 数据报表 | Excel | 每周一次 | I，自动发送 |

## Control（控制）

| 层级 | 控制点 | 手段 |
|------|--------|------|
| 事前 | 输入校验 | 敏感词过滤、完整性检查 |
| 事中 | 质量检查 | CHECK自检、超时控制 |
| 事后 | 效果评估 | 用户评分、数据追踪 |

## Adaptation（适配）

| 类型 | 学习对象 | 适配策略 |
|------|----------|----------|
| 上下文 | 当前对话 | 记住本次提到的偏好 |
| 用户偏好 | 历史选择 | 常选题材权重+30% |
| 场景 | 时间/节日 | 工作日vs周末不同推荐 |
```

## 自检清单

设计IOCA时，检查：

- [ ] 输入来源明确（固定/触发/人工）
- [ ] 输入有I-标准（完整性、准确性、及时性、格式）
- [ ] 输出格式具体（不是"文本"而是"Markdown表格"）
- [ ] 输出有O-标准（准确性、完整性、可用性、一致性）
- [ ] 有明确的质量评分标准（如>85分）
- [ ] 有三层控制机制（事前/事中/事后）
- [ ] 有适配机制（上下文/用户/场景）
