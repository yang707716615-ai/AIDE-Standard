# 02 - RACI-D标准（职责定义）

> **阅读时间**：10分钟  
> **目标**：学会用RACI-D分配人和AI的职责

---

## 什么是RACI-D？

**RACI-D** = **R**esponsible（执行）+ **A**ccountable（负责）+ **C**onsulted（咨询）+ **I**nformed（知情）+ **D**elegated（委托）

**起源**：来自项目管理经典的RACI矩阵，我们增加了D（Delegated），专门用于AI时代。

**核心作用**：
> 明确每个任务，人类和AI分别扮演什么角色，避免"AI该做的没做，人类不该做的忙死"。

---

## 五个角色详解

### 1. R - Responsible（执行）👤

**定义**：必须亲自执行任务的角色

**人类专属**：
- 最终审核（内容质量把关）
- 创意决策（方向性选择）
- 敏感操作（发布、签约、付款）
- 复杂判断（需要人类经验/情感的）

**示例**：
```
任务：发布视频到抖音
R（人类执行）：点击"发布"按钮
原因：发布行为涉及账号安全、内容责任，必须人工确认
```

---

### 2. A - Accountable（负责）✅

**定义**：对结果负全责的角色

**关键区别**：
- R是"动手做"，A是"拍板负责"
- 一个人可以既是R又是A
- 但A必须是人类（不能让AI背锅）

**示例**：
```
任务：AI生成营销文案
R（AI执行）：写文案
A（人类负责）：审核并批准发布
原因：如果文案惹了麻烦，是人类负责，不是AI
```

---

### 3. C - Consulted（咨询）💬

**定义**：提供专业建议，供决策者参考

**AI最佳角色**：
- 数据分析
- 方案生成
- 风险评估

**示例**：
```
任务：选择今天的视频选题
C（AI咨询）：AI分析热点，推荐5个选题+数据
人类决策：从5个中选1个，或提出新想法
```

---

### 4. I - Informed（知情）👁️

**定义**：事后被告知结果，不需要参与决策

**AI最佳角色**：
- 自动执行并记录
- 自动整理数据
- 自动生成报表

**示例**：
```
任务：记录视频发布数据
I（AI知情）：AI自动记录播放量、点赞数、评论数
人类查看：每周一看一次数据报表
```

---

### 5. D - Delegated（委托）🤖

**定义**：完全委托给AI自动执行，无需人类干预

**AI最佳角色**：
- 重复性任务
- 规则明确任务
- 低风险任务

**示例**：
```
任务：生成视频提示词
D（AI委托）：AI根据选题，自动生成3版提示词
人类：只管选题，不管生成过程
```

---

## RACI-D分配原则

### 原则1：A必须是人类
```
❌ 错误：让AI对结果负责
✅ 正确：人类永远对AI的输出负责
```

### 原则2：关键决策点必须有A
- 发布前（A审核）
- 签约前（A确认）
- 付款前（A批准）

### 原则3：从保守到激进
```
Phase 1（第1-10次）：多用C和R，少用D
Phase 2（第11-50次）：逐步增加D，减少R
Phase 3（第50+次）：大量用D，只在关键节点A
```

---

## 完整示例：视频创作AI员工

```markdown
### 任务1：每日选题
- **R（人类执行）**：最终选择1个选题
- **A（人类负责）**：对选题效果负责
- **C（AI咨询）**：AI分析热点，推荐5个选题
- **I（AI知情）**：AI自动记录选题历史
- **D（AI委托）**：AI自动监控热点榜单

### 任务2：提示词生成
- **R（人类执行）**：审核并确认最终提示词
- **A（人类负责）**：对视频质量负责
- **C（AI咨询）**：AI生成3版提示词供选择
- **I（AI知情）**：AI自动保存版本历史
- **D（AI委托）**：AI自动优化提示词

### 任务3：文案撰写
- **R（人类执行）**：最终发布前审核文案
- **A（人类负责）**：对文案合规性负责
- **C（AI咨询）**：AI生成4平台版本文案
- **I（AI知情）**：AI自动记录文案效果
- **D（AI委托）**：AI自动适配平台风格

### 任务4：视频发布
- **R（人类执行）**：点击发布按钮（必须人工）
- **A（人类负责）**：对发布内容负全责
- **C（AI咨询）**：AI推荐发布时间+标签
- **I（AI知情）**：AI自动记录发布数据
- **D（AI委托）**：无（发布必须人工）
```

---

## 自检清单

- [ ] 每个任务都有明确的A（负责人）
- [ ] A必须是人类，不能是AI
- [ ] 关键决策点有R或A
- [ ] 没有"全D"或"全R"的极端情况
- [ ] 分工符合"AI做重复，人类做决策"原则
