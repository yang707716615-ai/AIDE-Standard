# 设计阶段检查清单

> **使用时机**：设计AI员工时，填写完模板后自查  
> **目标**：确保设计完整，没有遗漏

---

## RACI-D检查

- [ ] **每个任务都有明确的A（负责人）**
  - 没有A的任务 = 没人负责，出问题找不到人
  
- [ ] **A必须是人类，不能是AI**
  - AI不能承担责任
  
- [ ] **关键决策点（发布/签约/付款）有R或A**
  - 不能全是C/I/D
  
- [ ] **C和D的任务有质量检查机制**
  - 定期检查C建议的质量
  - 定期抽查D执行的质量
  
- [ ] **没有"全D"或"全R"的极端情况**
  - 全D = 完全不管，会失控
  - 全R = 全人工，没用到AI
  
- [ ] **分工符合"AI做重复，人类做决策"原则**
  - AI：重复性、规则明确、数据驱动
  - 人类：创造性、责任重大、需要判断

---

## STATE检查

- [ ] **有明确的启动条件（什么触发IDLE→AUTO）**
  - 定时？指令？事件？
  
- [ ] **每个WAIT有超时机制（30-60秒）**
  - 避免无限等待
  - 超时后自动选择默认值
  
- [ ] **每个CHECK有明确的通过标准**
  - 质量评分？通过率？准确率？
  - 不通过怎么办？（重试/升级/停止）
  
- [ ] **每个HALT给人类清晰的操作指南**
  - 为什么停？
  - 人类需要做什么？
  - 做完后怎么继续？
  
- [ ] **有异常处理分支（不是只有主流程）**
  - CHECK不通过？
  - 超时？
  - 报错？
  
- [ ] **人类参与节点不超过30%**
  - 太繁琐用户会放弃
  - 理想：AI做70%，人类做30%
  
- [ ] **有明确的结束状态（回到IDLE）**
  - 任务完成标志
  - 资源释放

---

## IOCA检查

- [ ] **输入来源明确（固定/触发/人工）**
  - 每个输入都有明确来源
  - 人工输入有格式说明
  
- [ ] **输入有I-标准（完整性、准确性、及时性、格式）**
  - 输入质量如何保证？
  
- [ ] **输出格式具体（不是"文本"而是"Markdown表格"）**
  - 越具体越好
  
- [ ] **输出有O-标准（准确性、完整性、可用性、一致性）**
  - 质量怎么衡量？
  
- [ ] **有明确的质量评分标准（如>85分）**
  - 可量化
  
- [ ] **有三层控制机制（事前/事中/事后）**
  - 事前：输入校验
  - 事中：进度监控
  - 事后：效果评估
  
- [ ] **有适配机制（上下文/用户/场景）**
  - AI会学习吗？
  - 怎么越用越聪明？

---

## MBVE检查

- [ ] **有三层记忆（短期/中期/长期）**
  - 短期：当前对话
  - 中期：本次工作流
  - 长期：永久学习
  
- [ ] **记忆有明确的应用场景**
  - 记住什么？怎么用？
  
- [ ] **有行为边界（能/不能清单）**
  - ✅ 能做的
  - ❌ 不能做的
  
- [ ] **有ROI计算模型**
  - 节省时间价值？
  - AI成本？
  - ROI多少？
  
- [ ] **有QUICK质量评估**
  - Q/U/I/C/K五维评分
  
- [ ] **有三级进化路径（L1/L2/L3）**
  - L1新手：人工70%
  - L2熟练：人工30%
  - L3专家：人工5%
  
- [ ] **有升级和降级机制**
  - 什么条件升级？
  - 什么条件降级？

---

## 通用检查

- [ ] **目标明确**：这个AI员工解决什么问题？
- [ ] **范围清晰**：不做什么？（边界）
- [ ] **可测试**：怎么验证好不好用？
- [ ] **可迭代**：怎么根据反馈优化？
- [ ] **有案例**：参考了哪个案例？
- [ ] **有数据**：预期什么效果？（节省时间/提升产量）
