# 安全合规检查清单

> **使用时机**：AI员工设计完成，准备上线前  
> **目标**：确保安全合规，避免风险

---

## 数据安全

### 输入数据

- [ ] **敏感信息过滤**
  - 是否过滤了手机号、身份证号、银行卡号？
  - 是否过滤了密码、密钥等敏感信息？
  
- [ ] **个人隐私保护**
  - 是否涉及用户隐私数据？
  - 是否有数据脱敏处理？
  
- [ ] **数据来源合规**
  - 固定输入的数据来源是否合法？
  - 是否有数据使用授权？

### 输出数据

- [ ] **输出内容审核**
  - 是否有过滤机制防止输出敏感内容？
  - 是否有敏感词库？
  
- [ ] **数据存储安全**
  - 长期记忆（LTM）存储在哪里？
  - 是否有加密存储？
  - 访问权限如何控制？

---

## 内容合规

### 内容审核机制

- [ ] **事前审核（输入阶段）**
  - 输入内容是否经过敏感词过滤？
  - 是否有违法违规内容检测？
  
- [ ] **事中审核（处理阶段）**
  - CHECK节点是否有内容质量检查？
  - 是否有价值观对齐检查？
  
- [ ] **事后审核（输出阶段）**
  - 最终输出前是否有A（人类负责）审核？
  - 是否有发布前最后检查？

### 禁止内容清单

AI员工绝对不能生成：

- [ ] **违法内容**
  - 涉及国家安全
  - 涉及恐怖主义
  - 涉及违法犯罪
  
- [ ] **有害内容**
  - 暴力、血腥
  - 色情、低俗
  - 歧视、仇恨言论
  
- [ ] **虚假信息**
  - 造谣传谣
  - 虚假广告
  - 误导性内容
  
- [ ] **侵权内容**
  - 侵犯版权
  - 侵犯商标
  - 侵犯肖像权

---

## 人工确认节点（安全底线）

以下节点**必须**有A（人类负责）或R（人类执行）：

- [ ] **对外发布前**
  - 任何对外发布的内容必须人工最终确认
  - 包括但不限于：文章、视频、评论、回复
  
- [ ] **涉及交易/金钱**
  - 价格修改
  - 订单处理
  - 退款操作
  
- [ ] **涉及法律/合同**
  - 合同签署
  - 法律声明
  - 对外承诺
  
- [ ] **涉及用户权益**
  - 用户投诉处理
  - 用户数据操作
  - 用户协议变更

---

## 异常处理机制

### 异常检测

- [ ] **内容异常检测**
  - 输出内容包含敏感词？
  - 输出内容逻辑混乱？
  - 输出内容涉及禁止话题？
  
- [ ] **行为异常检测**
  - AI行为偏离预期？
  - 连续多次CHECK不通过？
  - 用户连续差评？
  
- [ ] **系统异常检测**
  - API调用失败？
  - 响应超时？
  - 数据丢失？

### 异常响应

- [ ] **自动响应**
  - 检测到异常自动HALT
  - 自动通知管理员
  - 自动记录异常日志
  
- [ ] **人工响应**
  - 异常后必须人工介入
  - 人工确认后才能继续
  - 人工记录处理结果

---

## 权限控制

### AI员工权限

- [ ] **明确权限边界**
  - AI能做什么？（D/C/I清单）
  - AI不能做什么？（必须R/A的清单）
  
- [ ] **权限最小化原则**
  - AI只有完成工作必需的权限
  - 没有多余权限（如删除数据、修改配置）
  
- [ ] **权限审计**
  - 定期审计AI权限
  - 记录权限使用情况
  - 及时发现权限滥用

### 用户权限

- [ ] **分级权限**
  - 管理员：全部权限
  - 操作员：有限权限
  - 访客：只读权限
  
- [ ] **权限分配记录**
  - 谁有权限？
  - 权限范围？
  - 分配时间？

---

## 审计与追溯

### 操作日志

- [ ] **记录所有操作**
  - AI执行了什么？
  - 人类确认了什么？
  - 什么时候执行的？
  
- [ ] **日志保留期限**
  - 至少保留90天
  - 重要操作永久保留
  
- [ ] **日志可追溯**
  - 能追溯到具体操作
  - 能追溯到操作人
  - 能追溯到决策依据

### 决策可追溯

- [ ] **RACI-D可追溯**
  - 每个决策都有A（负责人）
  - 能追溯到谁做的决策
  
- [ ] **STATE可追溯**
  - 完整的状态流转记录
  - 知道在哪个节点出了问题
  
- [ ] **IOCA可追溯**
  - 输入来源可追溯
  - 输出依据可追溯
  - 控制记录可追溯

## 应急预案

### 紧急情况

- [ ] **AI失控**
  - 立即停止AI运行
  - 切换到全人工模式
  - 分析原因，修复后再启动
  
- [ ] **数据泄露**
  - 立即停止相关功能
  - 通知受影响用户
  - 报告监管部门
  
- [ ] **内容违规**
  - 立即删除违规内容
  - 停止AI运行
  - 人工审查所有历史输出
